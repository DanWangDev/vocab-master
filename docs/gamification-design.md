## Problem Statement

Vocab Master has solid learning modes (flashcards, SRS, quizzes, spelling, timed, PvP, sentence building) but lacks emotional engagement. Kids use the app when told to but don't self-motivate. Parents can see quiz results but lack a "glanceable" proof-of-consistency view. The app works but doesn't *pull*.

The goal: make kids *want* to open the app. Make progress visible enough that a parent sees a 30-day heatmap and says "whoa." Make the moment-to-moment experience rewarding enough that a kid finishes dinner and asks "can I do my vocab?"

## What Makes This Cool

Three interlocking engagement loops, each targeting a different time horizon:

1. **Moment-to-moment: XP** — Every quiz answer, flashcard review, spelling word, and sentence built earns XP. Numbers go up. Dopamine flows. The kid feels every action is worth something.

2. **Session-to-session: Level-ups** — XP accumulates into levels with satisfying celebration animations. "What level are you?" becomes the playground question. Level badge shows next to username everywhere.

3. **Day-to-day: Streak + Heatmap + Rewards** — A pulsing flame on the dashboard. A GitHub-style 90-day heatmap that fills with green. Streak milestones (3, 7, 14, 30, 60, 90 days) unlock cosmetic rewards (avatar frames, dashboard colour themes). Breaking a streak means losing the frame. Loss aversion is the strongest retention lever.

## Constraints

- ~15 users (family + friends' kids), ages 9-11, UK 11+ exam prep
- SQLite on NAS via Docker — no need for horizontal scaling
- Solo developer — implementation must be achievable in focused sessions
- Must integrate with existing achievement, leaderboard, and SRS systems
- i18n required (English + Simplified Chinese)
- Mobile app exists (Expo) but web-first — mobile can follow later. Animation components (Framer Motion) are web-only; mobile will need `react-native-reanimated` equivalents.
- XP abuse prevention deferred — trusted user base of ~15 kids, no public API

## Premises

1. The learning modes are solid — the gap is emotional engagement, not more exercise types
2. Two audiences need two things: parents need glanceable proof (heatmap), kids need moment-to-moment reward (XP/levels)
3. The existing streak/stats/achievement infrastructure can be extended rather than rebuilt
4. Cosmetic rewards (avatar frames, dashboard themes) are sufficient — no need for a virtual currency shop or IAP complexity

## Approaches Considered

### Approach A: Streak & Heatmap Only
Minimal — just the visual progress layer. 90-day heatmap + streak flame on dashboard.
- **Effort:** S | **Risk:** Low
- **Pros:** Ships fast, solves parent visibility
- **Cons:** No moment-to-moment reward, no progression, no pull for kids
- **Completeness: 4/10**

### Approach B: Full Gamification Loop
Three-layer system: streak/heatmap + XP/levels + cosmetic streak rewards.
- **Effort:** L | **Risk:** Medium
- **Pros:** Complete engagement loop, social comparison, loss aversion
- **Cons:** Larger scope, XP balance tuning needed
- **Completeness: 9/10**

### Approach C: XP + Streak, No Cosmetics
Middle ground — streak + XP/levels but skip avatar frames and themes.
- **Effort:** M | **Risk:** Low-Medium
- **Pros:** Core loop captured, simpler
- **Cons:** No tangible rewards, no loss aversion from streak milestones
- **Completeness: 7/10**

## Recommended Approach

**Approach B: Full Gamification Loop.** The delta between C and B is small with AI-assisted development. The streak rewards create loss aversion ("I can't lose my 14-day avatar frame!") which is the single strongest retention lever. When the marginal cost is near-zero, always choose complete.

### Implementation Design

#### Layer 1: Streak Flame + 90-Day Heatmap

**Backend:**
- New endpoint `GET /api/stats/activity-heatmap?days=90&tz=` — returns array of `{ date, activityCount, types[] }` for each day. Accepts optional `tz` parameter (e.g., `Europe/London`); defaults to user's timezone from settings or UTC. All date comparisons normalize to the user's local date to avoid the 10pm-study-on-wrong-UTC-date problem.
- New endpoint `GET /api/stats/streak-details` — returns `{ currentStreak, longestStreak, streakStartDate, isActiveToday }`. Same timezone normalization as heatmap. This becomes the canonical streak endpoint. The existing `GET /api/stats` response will continue to include `current_streak` for backward compatibility, computed by the same logic.
- Data source: UNION query across `study_sessions`, `quiz_results`, `daily_challenges`, `exercise_results` (same full four-table UNION used in `dashboardService.calculateStreak()`)

**Frontend:**
- `StreakFlame.tsx` — SVG/CSS gradient flame (NOT an emoji) on dashboard
  - Tier 0 (0 days): No flame. Wind icon (💨) + "Start your streak!" text
  - Tier 1 (1-2 days): Small gray-blue flame, 24px. Color: gray-400. No animation
  - Tier 2 (3-6 days): Orange flame, 32px. Gradient: from-orange-400 to-amber-500. Gentle pulse scale(1→1.08) / 2s ease
  - Tier 3 (7-13 days): Larger flame, 40px. Gradient: from-orange-500 to-red-500. Steady glow (box-shadow pulse) / 1.5s
  - Tier 4 (14-29 days): Bright flame, 48px. Gradient: from-red-500 to-amber-400. Glow + 3-4 floating particles (small circles, random y-offset -20 to -40px, fade out)
  - Tier 5 (30+ days): Golden flame, 56px. Gradient: from-amber-400 to-yellow-300. Radiant glow + particles + subtle rotate(±5°) wobble
  - **Streak-break state:** Gray flame with text "Your N-day streak ended — start fresh today!" (encouraging tone, visible loss)
  - `aria-label="Current streak: N days"`
- `ActivityHeatmap.tsx` — CSS Grid (13 columns x 7 rows = 91 days)
  - Cell size: 12px desktop / 10px mobile, gap: 3px / 2px, border-radius: 2px
  - Color scale: gray-200 (0) → emerald-200 (1) → emerald-400 (2-3) → emerald-600 (4+)
  - Week labels (M/W/F) on left, month labels on top
  - Tap/focus a cell for day breakdown tooltip (which activities) — tooltip triggered by focus too for keyboard users
  - Responsive: full grid on desktop (fits in max-w-xl), overflow-x-auto with horizontal scroll on mobile (last 5 weeks visible)
  - Empty state: all gray cells + centered text "Your activity will appear here. Start studying to light it up!"
  - `role="img"` with `aria-label="Activity heatmap showing last 90 days"`

#### Layer 2: XP System + Levels

**Schema (new migration):**
```sql
CREATE TABLE user_xp (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL REFERENCES users(id),
  amount INTEGER NOT NULL,
  source TEXT NOT NULL,        -- 'quiz', 'challenge', 'study', 'exercise', 'pvp', 'streak_bonus'
  source_id INTEGER,           -- optional FK to the specific result
  earned_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_user_xp_user ON user_xp(user_id);
CREATE INDEX idx_user_xp_earned ON user_xp(earned_at);

ALTER TABLE user_stats ADD COLUMN total_xp INTEGER NOT NULL DEFAULT 0;
ALTER TABLE user_stats ADD COLUMN level INTEGER NOT NULL DEFAULT 1;
```

**XP Award Table:**

| Activity | XP | Bonus |
|----------|-----|-------|
| Quiz question (correct) | 10 | +5 if perfect quiz |
| Quiz question (incorrect) | 2 | (participation) |
| Daily challenge completed | 25 | +10 if > 80% |
| Flashcard reviewed (SRS) | 5 | +3 if quality >= 4 |
| Spelling word (correct) | 10 | +5 if first attempt |
| Sentence built (correct) | 10 | — |
| PvP challenge won | 30 | +15 if > 90% |
| PvP challenge lost | 10 | (participation) |
| Study session (per 5 min) | 5 | XP = floor(duration_minutes / 5) * 5. Minimum: 5 XP for any session >= 1 minute. Example: 3 min = 5 XP, 12 min = 10 XP, 27 min = 25 XP. Duration computed as `(julianday(end_time) - julianday(start_time)) * 86400000` ms from `study_sessions`. |
| Streak day bonus | streak_days * 2 | Awarded once per day on the user's first qualifying activity. Deduplicated via `SELECT 1 FROM user_xp WHERE user_id = ? AND source = 'streak_bonus' AND date(earned_at, tz_offset) = date('now', tz_offset) LIMIT 1`. A user on a 30-day streak earns 60 bonus XP on that day's first activity. |

**Level Thresholds:**

| Level | Total XP | Title |
|-------|----------|-------|
| 1 | 0 | Beginner |
| 2 | 100 | Novice |
| 3 | 300 | Apprentice |
| 4 | 600 | Scholar |
| 5 | 1000 | Wordsmith |
| 6 | 1500 | Linguist |
| 7 | 2200 | Lexicon |
| 8 | 3000 | Vocabularian |
| 9 | 4000 | Word Master |
| 10 | 5500 | Grand Master |

Levels above 10: Level N (N > 10) requires 5500 + (N - 10) * 2000 total XP. E.g., Level 11 = 7500, Level 12 = 9500.

**Backend:**
- New `xpService.ts` — `awardXp(userId, amount, source, sourceId)` uses a SQLite transaction wrapping both `INSERT INTO user_xp` and `UPDATE user_stats SET total_xp = total_xp + ?, level = ?`
- Returns `{ xpEarned, totalXp, level, leveledUp, newLevel }` so frontend can animate
- **XP award is best-effort.** If the transaction fails, the activity result is still saved. A warning is logged. XP can be reconciled later.
- **API response contract:** Each modified endpoint (quizResults, exercises, pvp, srs, challenges) appends an `xp: { earned, total, level, leveledUp }` object to its existing response payload. Frontend reads this from the response — no separate polling needed.
- Hook into existing result submission flows: `quizResults.ts`, `challenges.ts`, `exercises.ts`, `pvp.ts`, `srs.ts`. Future improvement: consolidate XP triggers into an event emitter pattern to reduce coupling. For initial implementation, direct calls in each route are acceptable given the small codebase.
- New endpoints: `GET /api/xp/history?limit=50`, `GET /api/xp/level-info`

**Frontend:**
- `XpPopup.tsx` — "+N XP" floating animation, Framer Motion
  - Position: floats up from the triggering element (answer button, submit button), not fixed-position
  - Animation: appear at element position, float up 40px, fade out over 1.2s
  - Font: Nunito Bold, color: amber-500, text-shadow for legibility
  - Queue: max 3 visible simultaneously, older ones merge/dismiss
  - No popup if xp.earned = 0
- `LevelUpCelebration.tsx` — full-screen overlay
  - Background: semi-transparent teal `bg-primary-900/80` (not generic black)
  - Confetti: uses app mode colors (teal/amber/coral), not generic rainbow
  - Level number: enters with `scale(0→1.2→1.0)` spring animation
  - Title: fades in 0.5s after number ("Level 5: Wordsmith")
  - Dismiss: tap/click or auto after 5s. Escape key dismisses
  - `role="dialog"`, `aria-modal="true"`, focus trap
- `LevelBadge.tsx` — small badge showing level number + title
  - Used in: dashboard header (replaces plain greeting), leaderboard rows, PvP opponent display
  - Mobile: compact "Lv.5" only. Desktop: "Lv.5 Wordsmith"
  - Loading state: "Lv. —"
- `XpProgressBar.tsx` — progress bar showing XP toward next level
  - `role="progressbar"` with `aria-valuenow`, `aria-valuemin`, `aria-valuemax`
  - Shows "340 / 1,000 XP" text. Empty: "0 / 100 XP" at Level 1
  - Bar color: gradient from-emerald-400 to-teal-500 (matches primary)
  - Loading state: empty bar with "-- / --"

#### Layer 3: Streak Milestone Rewards

**Schema (same migration 024):**
```sql
CREATE TABLE streak_rewards (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  streak_days INTEGER NOT NULL UNIQUE,   -- 3, 7, 14, 30, 60, 90
  reward_type TEXT NOT NULL,             -- 'avatar_frame', 'dashboard_theme'
  reward_slug TEXT NOT NULL UNIQUE,
  name TEXT NOT NULL,
  description TEXT NOT NULL
);

CREATE TABLE user_rewards (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL REFERENCES users(id),
  reward_id INTEGER NOT NULL REFERENCES streak_rewards(id),
  earned_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(user_id, reward_id)
);

ALTER TABLE user_settings ADD COLUMN active_avatar_frame TEXT DEFAULT NULL;
ALTER TABLE user_settings ADD COLUMN active_dashboard_theme TEXT DEFAULT NULL;
```

**Reward Milestones:**

| Streak | Reward | Name |
|--------|--------|------|
| 3 days | Avatar frame | Bronze Ring |
| 7 days | Dashboard theme | Ocean Blue |
| 14 days | Avatar frame | Silver Crown |
| 30 days | Dashboard theme | Sunset Gold |
| 60 days | Avatar frame | Golden Shield |
| 90 days | Dashboard theme | Platinum Aurora |

**Key mechanic: Rewards are "locked" if streak is broken.** The kid keeps the reward in their collection but it shows as locked/grayed if their current streak is below the threshold. They must rebuild the streak to re-activate it. This creates the loss aversion: "I can't skip today or I lose my Silver Crown!"

**Lock state is computed, not stored.** The `GET /api/rewards` endpoint compares the user's current streak against each reward's `streak_days` threshold to determine active/locked status at read time. No additional column needed in `user_rewards`.

**Backend:**
- Extend streak calculation to check milestones on each activity
- `rewardService.ts` — check/award rewards, get user rewards, set active frame/theme
- Endpoint `GET /api/rewards` — list all rewards with user's earned/active status
- Endpoint `PATCH /api/rewards/active` — set active frame/theme

**Frontend:**
- `RewardUnlockToast.tsx` — celebration toast when milestone reached
  - Reuse `AchievementUnlockedToast` pattern: `fixed top-4 left-1/2`, spring animation, 5s auto-dismiss
  - Gradient: from-emerald-400 to-teal-500 (differentiate from achievement gold)
  - Shows reward name + icon + "You earned [reward]!"
- `RewardGallery.tsx` — grid of all rewards
  - Layout: 2-col mobile, 3-col desktop, gap-3/gap-4
  - Active reward: full color, "Active" badge, tap to deactivate
  - Earned but locked: grayscale, lock icon overlay, "Rebuild to N days to unlock"
  - Unearned: dimmed, streak requirement shown ("Reach 14-day streak")
  - Empty state (none earned): "Complete a 3-day streak to earn your first reward!" with flame icon
  - Items use `button` role with `aria-label` including name + status
- `AvatarFrame.tsx` — renders frame ring around avatar (specs in Avatar Frame Specs above)
  - Used in: dashboard header, leaderboard rows, PvP opponent display
  - No active frame: render avatar at 36px without wrapper
- Dashboard theme applied via CSS variables + `data-theme` attribute on `<body>` (specs in Theme Palettes above)

#### Streak-Break Experience

When a kid's streak resets to 0 after missing a day:
- **Dashboard:** Gray flame (wind icon) with text: "Your N-day streak ended — start fresh today!" (encouraging, not punitive)
- **Locked rewards on dashboard:** Not shown on dashboard (only in Rewards page)
- **Rewards page:** Locked rewards show "Rebuild to N days to unlock again" (clear path to recovery)
- **First activity after break:** Normal XP popup. Streak shows "1 day" immediately. Flame lights back up. Recovery feels instant.

## Open Questions

1. **XP for historical activity?** Should we backfill XP for existing quiz/challenge/study data so kids don't start at Level 1 after months of use? (Recommendation: defer — backfill logic for bonuses like "perfect quiz" across all activity tables is complex and error-prone. Instead, start everyone at Level 1 and let them earn their way up. The fresh start is actually exciting: "race to Level 10!")
2. **Streak freeze?** Should there be a "streak freeze" item (like Duolingo) that protects against one missed day? Could be earned at certain XP thresholds. (Recommendation: defer to a follow-up — keep it simple first)
3. **Mobile parity?** The Expo app doesn't have many of these features yet. Should the gamification system be web-only initially? (Recommendation: yes, web-first, mobile follows)
4. **Leaderboard integration?** Should XP replace or supplement the existing leaderboard score formula? (Recommendation: supplement — add XP as a visible stat on leaderboard rows but keep the existing score formula)
5. **Streak-at-risk notification?** Consider a daily reminder notification (push or in-app) at a configurable time if the user has not yet completed any activity for the day. This amplifies the loss aversion mechanic — a child can't feel the loss of their Silver Crown if they don't know it's at risk. (Recommendation: add in Phase C alongside the reward system)

## Success Criteria

- Kids open the app voluntarily at least 3x/week (currently: only when reminded)
- At least 5 of 15 users maintain a 7+ day streak within the first month
- Parents mention the heatmap or streak unprompted ("I saw they've been studying every day!")
- At least one kid asks about levels or frames ("what level are you?")
